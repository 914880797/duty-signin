# 页面功能说明

## index.html — 打卡首页

- **功能**：值班人员每日签到打卡的主入口
- **核心流程**：
  1. 自动显示当前北京时间
  2. 用户输入值班姓名，系统自动匹配当前时段
  3. 匹配成功后提交打卡记录
  4. 显示打卡成功/失败的提示
- **其他功能**：
  - 公告跑马灯：滚动显示管理员发布的最新公告
  - 导航链接：进入排班查询、打卡查询、补卡页面
  - 管理员入口：跳转到 admin 管理后台

## export.html — 导出预览

- **功能**：值班数据的汇总统计与导出
- **核心功能**：
  - 按日期范围展示所有人员的值班汇总表
  - 统计总人数、累计值班次数、人均值班次数
  - 搜索和筛选特定人员
  - 导出为 Excel 文件
- **表格特性**：
  - 行：人员姓名 + 分组
  - 列：日期 + 时段组合
  - 补卡记录用橙色 `补` 字标记
  - 移动端：姓名列 sticky 固定，统计列随日期滚动

## query.html — 排班查询

- **功能**：按条件查询当前的排班安排
- **筛选维度**：
  - 按值班时段筛选
  - 按分组筛选
  - 按关键词搜索姓名
- **展示内容**：显示筛选后的排班人员列表，含时段、姓名、分组信息

## makeup.html — 补卡

- **功能**：漏打卡人员补录打卡记录
- **核心流程**：
  1. 自动加载当日所有排班了但未打卡的记录（漏打卡列表）
  2. 支持按姓名搜索漏打卡记录
  3. 点击补卡按钮为该人员补录一条打卡记录
  4. 下方展示已补卡记录
- **规则**：只有已排班但未打卡的人才可补卡，未排班的人员不显示

## admin.html — 管理后台

- **功能**：系统管理员的后台管理界面
- **Tab 页面**：
  1. **排班管理**：搜索/查看/新增/修改/删除当日排班，支持一键批量删除
  2. **分组管理**：增删改查值班分组，调整分组排序
  3. **打卡设置**：配置允许打卡的人员白名单
  4. **时段配置**：自定义值班时段（增删改查）、默认时段一键初始化
  5. **公告管理**：发布/删除首页滚动公告
  6. **批量排班**：单时段批量添加人员 + 智能导入（粘贴中文排班文本自动解析）
- **安全**：需要管理员账号密码登录
